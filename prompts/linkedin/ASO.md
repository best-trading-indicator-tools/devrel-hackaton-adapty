# ASO Playbook

Use this guide when the post is a SOIS post touching ASO, store visibility, keyword strategy, app page conversion, ratings, or localization.

Operating contract:
- Treat this as strategy guidance, not a source of hard facts.
- Keep claims practical and scoped to app category, geo, and traffic mix.
- Do not copy lines from this file into final output.
- When giving numbers, use either user-provided metrics or clearly framed heuristics.

Practical structure:
1. Start with one uncomfortable but useful truth about ASO.
2. Name the diagnostic signal teams should check first.
3. Explain the mechanism behind the metric.
4. Give one to three actions teams can run this week.
5. Add caveats by market, app stage, and platform.

Diagnostic order:
- Validate intent match before chasing volume keywords.
- Fix screenshot and first impression clarity before micro-tuning long descriptions.
- Repair ratings and review motion if trust is the bottleneck.
- Run localization after the main market baseline is stable.

Signals and mechanics:
- High impressions plus low install conversion usually indicates listing friction.
- Strong conversion plus weak impressions usually indicates discoverability or ranking limits.
- Ranking wins with weak retention can mean low-intent acquisition.
- Ratings momentum often affects top-of-funnel trust and conversion quality.

Tactical patterns to surface:
- One primary value proposition per screenshot beats feature dumping.
- First two screenshots should explain outcome in plain language.
- Metadata should mirror user intent words, not internal product jargon.
- Localization should adapt meaning and examples, not just translate text.
- A/B tests should isolate one major variable and run long enough for stable reads.

SOIS quality bar:
- Include at least one concrete metric, threshold, or example scenario.
- Include one boundary condition such as "for utility apps" or "in paid-heavy traffic."
- End with a practical next step, checklist, or experiment idea.
